# Interview Checklist

Best practices on how to behave during a coding interview.

## Before

- [ ] Read, prepare _self-introduction_
- [ ] Read, prepare _final questions to ask_
- [ ] Prepare **pen and paper** (_notes, visualizing stuff_)
- [ ] Use earphones, make sure to be in a quiet environment
- [ ] Check that **internet, webcam, audio** are working
- [ ] Familiarize with, set up shortcuts coding environment (_Coderpad, CodePen_)
- [ ] Turn off webcam if possible (_less distractions, easier on the network bandwidth_)

**Note:** Learn about whiteboard space management for onsite whiteboarding interviews. Request for the option to use Zoom/Google Meet/Hangouts instead of a phone call for phone screen interview.

## During

- [ ] Introduce yourself in a few sentences (< minute or 2)
- [ ] Do not jump into coding right away/before the interviewer gives you the green light
- [ ] **Clarify problem** (_assumptions, input, range, format_)
  - [ ] Paraphrase and repeat the question (understand what they're asking)
  - [ ] **Clarify assumptions** (e.g. _Can you modify the original array / graph / data structure in any way?_, _How is the input stored?_, _If you are given a dictionary of words, is it a list of strings or a Trie?_, _Is the input array sorted?_)
  - [ ] **Clarify input value range** (Inputs: how big and what is the range?)
  - [ ] **Clarify input value format** (Values: Negative? Floating points? Empty? Null? Duplicates? Extremely large?)
  - [ ] **Work through a simplified example** to make you understood the question
- [ ] **Work out and optimize your approach**
  - [ ] Make some time for a 2-way discussion on the correct approach to take for the question
  - [ ] If you get stuck on the approach or optimization, use the structured approach to find a good approach
  - [ ] Explain a few approaches that you could take at a high level
  - [ ] Discuss the tradeoffs of each approach
  - [ ] State and explain the time and space complexity of your proposed approach(es)
  - [ ] Identify repeated/duplicated/overlapping computations and reduce them via caching
  - [ ] _Do not ignore any piece of information given_
  - [ ] _Do not appear unsure about your approach or analysis_
- [ ] **Code out your solution while talking through it**
